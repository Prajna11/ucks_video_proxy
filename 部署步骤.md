# Cloudflare Pages 部署步骤

## 方法一：通过 Git 仓库部署（推荐）

### 1. 初始化 Git 仓库并推送到 GitHub

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: Video proxy service"

# 添加远程仓库（替换为你的 GitHub 仓库地址）
git remote add origin https://github.com/你的用户名/ucks_video_proxy.git

# 推送到 GitHub
git push -u origin main
```

### 2. 在 Cloudflare Pages 部署

#### 中文界面操作流程：

1. **访问 Cloudflare 控制台**
   - 打开 https://dash.cloudflare.com/
   - 登录你的 Cloudflare 账号

2. **进入 Workers 和 Pages**
   - 方法一：在左侧菜单找到 **网络** 或向下滚动找到 **Workers 和 Pages**
   - 方法二：直接访问 https://dash.cloudflare.com/?to=/:account/workers-and-pages
   - 方法三：点击左上角 Cloudflare 图标，在顶部导航栏找到 **Workers 和 Pages**

3. **创建应用程序**
   - 点击右上角蓝色按钮 **创建应用程序** 或 **创建**

4. **选择 Pages**
   - 在弹出的页面中，选择 **Pages** 选项卡

5. **连接到 Git**
   - 点击 **连接到 Git** 按钮

6. **授权 GitHub**
   - 如果是第一次使用，会提示授权 GitHub
   - 点击 **连接 GitHub**
   - 在弹出的 GitHub 页面中点击 **授权 Cloudflare Pages**

7. **选择仓库**
   - 在仓库列表中找到 `ucks_video_proxy`
   - 点击仓库右侧的 **开始设置** 按钮

8. **配置构建设置**
   - **项目名称**: `ucks-video-proxy`（可自定义）
   - **生产分支**: `main`
   - **框架预设**: 选择 `无`
   - **构建命令**: 留空（不填）
   - **构建输出目录**: 输入 `/`
   - **环境变量**: 不需要添加

9. **保存并部署**
   - 点击底部 **保存并部署** 按钮

10. **等待部署完成**
    - 部署过程大约需要 30 秒到 1 分钟
    - 部署成功后会显示绿色的 **成功** 标志
    - 你会看到分配的域名，例如：`ucks-video-proxy.pages.dev`

### 3. 等待部署完成

部署完成后，你会得到一个 URL，类似：
```
https://ucks-video-proxy.pages.dev
```

### 4. 使用代理服务

在 ucks.cn 网站中使用：
```html
<video controls>
  <source src="https://ucks-video-proxy.pages.dev/proxy?url=视频URL" type="video/mp4">
</video>
```

# Vercel 部署步骤

## 1. 从 GitHub 导入项目

1. 登录 Vercel 控制台。
2. 点击 **Add New...** -> **Project**。
3. 选择你的 GitHub 仓库并导入。
4. 构建设置保持默认即可（本项目无构建步骤）。
5. 点击 **Deploy**。

## 2. 访问方式

部署成功后：

- 默认 API 地址：`https://你的域名/api/proxy?url=...`
- 如果使用仓库内置的 `vercel.json`，也可以：
  - `https://你的域名/?url=...`
  - `https://你的域名/proxy?url=...`

## 方法二：通过 Wrangler CLI 部署

### 1. 安装 Wrangler CLI

```bash
npm install -g wrangler
```

### 2. 登录 Cloudflare

```bash
wrangler login
```

这会打开浏览器，授权 Wrangler 访问你的 Cloudflare 账号。

### 3. 部署项目

```bash
# 在项目目录下执行
wrangler pages deploy . --project-name=ucks-video-proxy
```

### 4. 后续更新

每次修改代码后，重新执行：
```bash
wrangler pages deploy . --project-name=ucks-video-proxy
```

## 配置自定义域名（可选）

1. 在 Cloudflare Pages 项目页面
2. 点击 **Custom domains**
3. 点击 **Set up a custom domain**
4. 输入你的域名（如 `proxy.ucks.cn`）
5. 按照提示添加 DNS 记录
6. 等待 DNS 生效

## 测试代理服务

部署完成后，测试 URL：
```
https://ucks-video-proxy.pages.dev/proxy?url=https://你的视频URL
```

在浏览器中直接访问会提示 Referer 错误（这是正常的，因为需要从 ucks.cn 引用）。

在 ucks.cn 网站中嵌入视频即可正常播放。

## 常见问题

### 1. 部署失败
- 检查 Git 仓库是否正确推送
- 确保 `functions/proxy.js` 文件路径正确

### 2. 视频无法播放
- 检查视频源域名是否在 `ALLOWED_VIDEO_DOMAINS` 中
- 检查 ucks.cn 是否在 `ALLOWED_REFERRER_DOMAINS` 中
- 打开浏览器控制台查看错误信息

### 3. 更新代码
- Git 方式：推送新代码到 GitHub，Cloudflare 会自动重新部署
- Wrangler 方式：重新执行 `wrangler pages deploy .`
